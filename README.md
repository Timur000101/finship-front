Техническое задание

Адаптация дизайна к текущему фронтенду

Текущая ситуация — есть фронтенд написанный на vue ( spa приложение ). Необходимо подогнать текущие компоненты (экраны) к новому дизайну. 


Задачи:

    1. Сверстать и подогнать текущие экраны и компоненты максимально точно под макеты дизайна в psd. Основная задача верстка уже в компонентах vue.
    2. Навести красоту, подобрать иконки, подровнять отступы 

Требования:

    1. Использовать только vue, vuex,vue-router,bootstrap-vue, использовать только классы bootstrap и его интерфейсы, scss для полей ввода 
    2. Иконки использовать только fontawesome. https://use.fontawesome.com/releases/v5.3.1/css/all.css
    3. URL для статики определять как /static/. Пример: /static/image.png
       
    4. Методы писать не нужно, можно просто создать пустой метод и установить его на событие тега к примеру click
    5. На выходе должно получиться тоже самое только с новым дизайном


Входные данные:

    1. Вы получаете доступ к репозиторию фронтенда, необходимо склонировать его! Все изменения, комиты, результаты работы отправлять в репозиторий.
    2. Фронтенд сделан на Vue. Используется также Vuex, vue-router, vue-boostrap и другие компоненты. Собирается с помощью vue-cli.
    3. Иконки используются fontawesome

Инструкция для фрилансера и Алгоритм как сделать это быстрее и эффективнее:

    1. Необходима верстка и подгонка текущих компонентов написанных на vue при использовании интерфейсов bootstrap-vue
    2. Каждый компонент имеет верстку между тегами template. Вырезаем этот код, переносим в свою настроенную среду разработки программного обеспечения
    3. Подгоняем верстку под дизайн, затем тестируем верстку на разных разрешениях (через консоль разработчика на разных устройствах) 
    4. Также нужно протестировать что сообщения ошибок ( ошибки валидации, ошибки с сервера), заключенное между тегом b-form-invalid-feedback корректно отображаются 
    5. Копируем обратно оттестируемую верстку обратно в репозиторий и пушим на сервер
    6. Писать методы не нужно, просто делаем пустыми, к примеру назначили пустой метод на событие @click кнопки
    
Настройка, как сейчас используется:

    1. Склонируйте репозиторий на свою локальную машину
    2. Установить node 
    3. Установить сборщик vue-cli
    4. Скачайте все пакеты командой npm install
    5. Соберите фронтенд командой npm run build
    6. В данный момент используется команада npm run build и сервер nginx для раздачи index.html и статики типа app.js
    7. nginx-finship.conf - настройка для nginx, не забудьте переписать пути внутри nginx-finship.conf затем скопировать в директорию /etc/nginx/sites-enabled и перезапустить nginx командой sudo service nginx restart
